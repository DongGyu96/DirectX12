DXGI_MODE_SCANLINE_ORDER_UNSPECIFIED
그래픽 하드웨어가 알아서 순서대로 그리도록 하는 것 ( 가장 최선의 선택으로 )
프로그래시브 - 홀수,짝수번째 필드중 두 필드를 한번에
어퍼 로우 필드 - 홀수 or 짝수번째 필드를 먼저 그린다

DXGI_FORMAT : 픽셀의 색상 형식을 표현
D3D, DXGI 모두 사용됨 텍스처들의 픽셀 형식에도 사용함
UNKNOWN은 쓸때마다 다르게 써도 된다는 의미
끝이 _FLOAT라면 실수 색상값을 32비트 or 16비트 실수로 표현할 수 있음
끝이 _SINT라면 음수가 존재하는 int형으로 되어있음
끝이 _UINT라면 음수가 존재하지 않는 unsigned int
끝이 _SNORM, _UNORM은 각각 signed, unsigned 형태의 Normalize된 값
[-1, 1], [0, 1] 보통 UNORM이 가장 많이 사용 ( 음수가 없기 때문 0부터 1까지 )
끝이 _TYPELESS는 형식이 없음 (void) 타입을 정해두는것이 아님


계단현상을 줄이는 법
픽셀을 줄인다 그러나 무한히 줄일 수는 없다
계단현상이라는 것은 디지털 출력장치에서는 궁극적으로 없앨 수 없음
계단 현상 제거 기법
슈퍼 샘플링 / 다중 샘플링
픽셀들의 집합에서 몇개를 모아서(샘플링) 필터링 하는것
가장 빠른 필터링은 평균 내는 것

DXGI_SAMPLE_DESC - 다중 샘플링의 품질 지정
Quality의 경우는 그래픽 하드웨어마다 최대 레벨이 정해져있음
그래픽 하드웨어가 지원하는 레벨을 알아보려면
ID3D12Device::CheckFeatureSupport 함수를 통해 알 수 있음 54p
최소 Count = 4, Quality = 1 를 할 수 있어야함 

슈퍼 샘플링 (4x)
출력할 전면버퍼의 해상도보다 4배 큰 후면버퍼와 깊이 버퍼를 만들어서 렌더링
4개의 픽셀이 하나의 픽셀로 옮겨지는 작업을 수행하게 됨
프리젠트할 때 후면 버퍼를 샘플링(평균을 내는 등)하여 픽셀 색상을 구함
4배보다 더 크게 만들어서 연산할 수도 있음
그러나 픽셀 수가 4배가 늘어나기 때문에 렌더링 시간도 4배가 늘어남
그래서 그래픽 하드웨어가 이 샘플링 방법을 지원해야함

MSAA :4X
슈퍼 샘플링처럼 후면버퍼와 깊이버퍼를 화면보다 4배 크게 생성
슈퍼 샘플링은 픽셀의 모든 색상을 계산하지만 이 방법은 픽셀마다 한번의 색상을
계산함
서브 픽셀의 색은 가시성과 포함여부에 따라 결정
가시성 : 각 서브 픽셀이 보이는지 카메라에서
포함여부 : 각 서브 픽셀의 중심이 다각형 내부에 존재하는지?


DXGI_USAGE BufferUsage;
보통 DXGI_USAGE_RENDER_TARGET_OUTPUT을 사용함
만들어진 스왑체인은 Direct3D가 렌더 타겟으로 쓰일것이라는 의미
D3D가 직접 렌더링을 할 것이 아니라면 안만들어도 됨
BufferCount - 후면 버퍼의 개수

DXGI_SWAP_EFFECT SwapEffect;
모든 디지털 장치는 메모리에 어떤 값을 유지시키려고 하면 자원이 소모됨
일반적으로 후면에서 전면으로 스와핑을 하고난다면 전면버퍼에 있는 내용을
굳이 유지할 필요가 없기 때문에 ( 어짜피 다시 그려야 하기 때문에 )
DXGI_SWAP_EFFECT_DISCARD로 지정하여 전면의 내용을 폐기함
후면버퍼가 여러개가 있다면 SEQUENTIAL로 차례대로 스왑하게함

Flags
DXGI_USAGE와 다른 이유는 데이터 타입이 무조건 enum중 하나여야 하지만
UINT인 Flags는 꼭 enum이 아니여도 된다는 뜻
또한 비트와이즈(연산자) | 을 사용해서 여러개의 옵션을 지정할 수 있다는 의미
DXGI_SWAP_CHAIN_FLAG_NONPREPOTATED 컴퓨터에선 불필요 ( 피벗 )
DXGI_SWAP_CHAIN_FLAG_ALLOW_MODE_SWITCH
바탕화면의 디스플레이 모드를 바꿀 수 있는 스왑체인 모드
바탕화면의 해상도를 응용프로그램이 바꿀 수 있음
DXGI_SWAP_CHAIN_FLAG_GDI_COMPATIVLE : Win API창을 사용 가능
